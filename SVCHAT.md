# SVchat — где что находится и последние изменения

> Обновлено: 2026-06-29 · Прод-версия: **v194** · В ветке готова: **v195**

SVchat — мессенджер-PWA (звонки, личные чаты, группы, опросы, медиа, push).
Один inline-React внутри `index.html` + Node.js-сервер на Socket.IO.

---

## 1. Где размещён SVchat (инфраструктура)

| Что | Где | Назначение |
|-----|-----|-----------|
| **Приложение + сервер** | **Render.com** (`svchat-server`) | Node.js (`server.js`), отдаёт `index.html`, Socket.IO, push. Не блокируется в РФ. |
| **База данных** | **PostgreSQL на Render** | Пользователи, аккаунты, история, участники комнат, push-подписки. `DATABASE_URL`. |
| **Медиа (фото/видео)** | **Timeweb S3** (`s3.twcstorage.ru`, бакет `svchat-media`, регион `ru-1`) | Файлы в объектном хранилище; в базе хранится только ключ. Тариф ~79 ₽/мес. |
| **Звонки (TURN)** | **metered.ca** | Ретрансляция WebRTC, чтобы звонки работали за NAT/мобильными сетями. |
| **Домен + DNS** | **Cloudflare** (`svchat24.ru`) | `svchat24.ru` и `www` → CNAME на `svchat-server.onrender.com` (**DNS only**, не proxied). |
| **Исходный код** | **GitHub** `avgur264-bot/svchat-server` | Render деплоит из ветки `main` автоматически. |
| **Android-приложение** | `svchat-native/` (Capacitor) | WebView-обёртка, грузит `https://svchat24.ru`. APK собирается локально. |
| **iOS-приложение** | `svchat-native/ios/` (Capacitor) | Проект есть, но собрать .ipa можно только на Mac с Xcode. |

### Поток данных
- Клиент (браузер/PWA/Android-WebView) ↔ **Render** по Socket.IO (`polling`+`websocket`).
- Сообщение с медиа → сервер кладёт файл в **Timeweb S3**, в Postgres пишет ключ → при открытии media подтягивается из S3.
- Звонок: сигналинг через Render, аудио/видео — напрямую P2P или через **metered.ca** TURN.
- Push — Web Push (VAPID) с Render на подписки в базе.

### Важно про Россию
- Прямой WebSocket к хостингам РФ-операторы душат DPI; **Cloudflare-прокси (оранжевое облако) в РФ заблокирован**.
- Поэтому приложение живёт на **Render** (его IP в РФ не блокируются), а Cloudflare используется только как DNS (CNAME, серое облако).
- Российский VPS (Timeweb) пробовали — на мобильных не открывался, **удалён**. Осталось только S3 от Timeweb.

### Переменные окружения (значения — в панели Render, не в коде)
`DATABASE_URL`, `S3_ENDPOINT`, `S3_BUCKET`, `S3_REGION`, `S3_ACCESS_KEY`, `S3_SECRET_KEY`,
`TURN_USERNAME`, `TURN_CREDENTIAL`, `VAPID_PUBLIC`, `VAPID_PRIVATE`, `OWNER_KEY`.

## 2. Как устроена сборка/версии

- Версия клиента: `window.__svBuild=N` в `index.html` **и** `const CLIENT_BUILD = N` в `server.js` — должны совпадать (по ним идёт авто-обновление клиента).
- При **любом** изменении `index.html` нужно пересчитать **CSP-хэш** module-скрипта (sha256, base64) — он встречается в `server.js` **дважды**.
- Проверка прод-версии: `https://svchat24.ru/stats` → `{"ver":N,"client":N}`. Здоровье: `/health`.
- Деплой: пуш в ветку → слияние в `main` (`git push origin <ветка>:main`) → Render собирает сам.

## 3. Последние изменения

### v195 — нативный звонок на Android  *(готово в ветке, ждёт деплоя)*
- Новый Capacitor-плагин **CallRing** (`svchat-native/.../CallRingPlugin.java`): по входящему звонку
  проигрывает **системный рингтон по кругу**, **вибрацию** и показывает **полноэкранное уведомление**
  категории «звонок» (видно поверх заблокированного экрана). Авто-стоп через 45 c; `stop()` при ответе/отбое.
- `MainActivity.java` регистрирует плагин; в манифест добавлены `VIBRATE`, `WAKE_LOCK`,
  `USE_FULL_SCREEN_INTENT`, `POST_NOTIFICATIONS`.
- Веб определяет запуск внутри Capacitor и зовёт нативный звонок; в обычном браузере — прежний веб-звук.
- APK: `svchat-native/SVchat-v195-debug.apk` (versionName 1.0.195).
- **Граница:** звонит, пока процесс приложения жив (в т.ч. фон/гашёный экран). Если приложение
  полностью выгружено из памяти — нужен FCM-пуш (Firebase), пока не сделано.

### v194 — видимость только своих чатов и групп  *(на проде)*
- Клиент при синхронизации списка (`my_rooms`) теперь сверяется с серверным членством:
  локальные комнаты, которых нет в ответе сервера (кик из группы, удаление собеседником, чужие/призрачные
  записи), **убираются** из списка. Открытый сейчас чат защищён от случайного удаления.
- Итог: каждый видит только те контакты и группы, **где он реально участник**.

### v193 — звонок звучит, даже если собеседник в другом чате  *(на проде)*
- Сервер при звонке шлёт живое событие `incoming_call` на **все** сокеты собеседника, а не только push при оффлайне.
- Клиент показывает баннер «Входящий звонок» с кнопками **Ответить / ✕**, проигрывает звук и вибрацию,
  повторяет звонок, пока звонящий звонит; «Ответить» открывает личный чат.

### v192 — усиление входящего  *(на проде)*
- Повтор push-уведомления о звонке + сильная вибрация `[400,200,...]` + `requireInteraction`.

### Ранее
- **Медиа в Timeweb S3** — фото/видео в объектное хранилище, база не растёт.
- v186–v190 — опросы в группах, тип ответа опроса, редизайн экрана звонка, рингтон/гудки, self-video на весь экран.

## 4. iPhone — статус

- Сборка iOS возможна **только на Mac с Xcode** (.ipa здесь собрать нельзя).
- Настоящий входящий звонок при заблокированном экране на iOS возможен **только через CallKit + VoIP-пуш (APNs)**;
  требует Mac + **платный Apple Developer** ($99/год) + доработку сервера. Код можно написать заранее.
- Сейчас на iPhone работает **PWA**: открыть `svchat24.ru` в Safari → «Поделиться» → «На экран Домой».
  Звук звонка есть, пока приложение открыто (передний план).

## 5. Полезные команды

```bash
# Текущая версия и здоровье прода
curl -s https://svchat24.ru/stats
curl -s https://svchat24.ru/health

# Сборка Android APK
cd svchat-native/android && ./gradlew assembleDebug
# готовый файл: app/build/outputs/apk/debug/app-debug.apk

# Деплой: слияние ветки в main (Render задеплоит сам)
git push origin claude/app-store-delete-account-7xmdw7:main
```
