# SVchat — E2E тесты (Playwright)

Автоматические сквозные тесты: открывают приложение в браузере, входят, шлют
сообщения, проверяют доставку по WebSocket и т.д.

## Установка (один раз)

```bash
cd svchat-e2e
npm install
npx playwright install chromium
```

## Запуск

```bash
npm test            # все тесты против прода
npm run test:smoke  # только стабильные (без WebSocket) — быстрый «жив ли сайт»
npm run test:realtime  # только realtime (отправка/доставка/файлы) — нужна реальная сеть
npm run test:headed    # с видимым браузером
npm run test:report    # открыть HTML-отчёт после прогона
```

Против локального сервера:

```bash
SVCHAT_URL=http://localhost:8080 npm test
```

## CI (GitHub Actions)

Workflow `.github/workflows/e2e.yml` запускается на каждый push в `main`, по
расписанию и вручную (вкладка **Actions → E2E Tests → Run workflow**).

- **Smoke-тесты** (1, 2, 3, 8) — обязательный гейт: если упали, сборка красная.
- **Realtime-тесты** (`@realtime`) — информативные, помечены `continue-on-error`:
  они прогоняются и видны в отчёте, но **не валят сборку**, т.к. против живого
  сервера realtime бывает нестабилен (холодный старт, переподключения). Их
  удобнее гонять локально (`npm run test:realtime`), где сеть реальная.

## Что проверяется

| # | Тест | Нужен WebSocket |
|---|------|:---:|
| 1 | Health-check (`/health`: ok, db, push) | нет |
| 2 | PWA грузится, `__svBuild` актуальный, без критичных ошибок | нет |
| 3 | Вход выполнен — пользователь внутри приложения | нет |
| 4 | Отправка сообщения в групповой чат | да |
| 5 | Доставка второму пользователю в реальном времени | да |
| 6 | Превью последнего сообщения в списке чатов | да |
| 7 | Авто-переподключение после обрыва сети | да |
| 8 | Мобильная вёрстка — нет горизонтального скролла | нет |
| 9 | Статус «прочитано» ✓✓ (Боб открыл чат → синие галочки) | да |
| 10 | Личный чат + индикатор шифрования 🔒 (E2E) | да |
| 11 | Отправка файла — карточка с именем появляется | да |

## Как это работает

- **Вход без онбординга:** тест кладёт в `localStorage` `chatapp_authed` и
  `svchat_profile` — сервер доверяет `userId` (TOFU), поэтому полноценный логин
  через телефон/почту в тестах не нужен.
- **Тест двух пользователей (#5):** открывает два независимых браузерных
  контекста (Алиса и Боб), оба входят в одну открытую группу; Алиса шлёт —
  ждём появления у Боба. Это прямая проверка Socket.IO.
- **Холодный старт Render:** перед отправкой тесты ждут установления сокета
  (баннер «сервер просыпается…» исчезает), таймаут до 70 сек.

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `SVCHAT_URL` | адрес приложения (по умолчанию прод) |
| `PW_INSECURE=1` | игнорировать ошибки TLS (за корпоративным прокси) |
| `PW_EXEC=/путь/к/chrome` | свой бинарь Chromium |

## Примечание про тесты, требующие WebSocket

Тесты 4–7, 9–11 проверяют realtime (Socket.IO) и проходят там, где соединение
`wss://` не блокируется (ваш компьютер, GitHub Actions). За TLS-перехватывающим
прокси WebSocket может не подняться — тогда используйте `PW_INSECURE=1` или
запускайте из обычной сети.

Тест 10 (🔒) требует, чтобы оба клиента обменялись E2E-ключами — он открывает одну
личку из двух контекстов и ждёт появления индикатора шифрования.
